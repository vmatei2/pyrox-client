Plan: Build-gate integrity guard for the DuckDB ingest (#18)¶

Repo: pyrox-client Target file: scripts/ingest_duckdb_from_s3.py Test file: tests/test_ingest_build_gate.py (new) Type: AFK — implement and merge without human interaction, provided all tests pass.

Background (read this first, do not skip)¶

The upstream scraper (hyrox_analysis repo) had a bug nicknamed "phantom duplicate races." The mikatiming results server returns HTTP 200 and silently substitutes a default event when asked for an event id it cannot resolve. The old scraper accepted that substituted page, so a single athlete's splits got written into many divisions/venues they never actually raced. Two fingerprints identify the corruption:

  1. Fan-out — the same roster duplicated across divisions, so within one (location, year, division) group row_count / distinct_athlete_names is well above 1.0 (clean singles is ~1.0; corrupt groups were ≥2.0).
  2. Zero roxzone — the substituted page came back with empty transition (Roxzone) fields, so nearly every finisher in a corrupt group has roxzone_time == 0.

The scraper has since been fixed and now validates at write time. This task adds the mirror check at the consumer so corrupt data — whether a leftover historical partition or a future regression — can never silently reach the queryable DuckDB warehouse. The build must fail loudly instead.

This is defense in depth, not a replacement for the upstream fix. Do not try to "clean" or repair data here. The only job is: detect the fingerprint, refuse to build, report exactly which event groups tripped it.

What to build¶

A read-only integrity check that runs over the assembled race_results table after it is created but before any downstream tables are built, and aborts the ingest if any event group shows the phantom fingerprint — unless an explicit override flag is passed.

Step 1 — Add the integrity check function¶

In scripts/ingest_duckdb_from_s3.py, add a module-level function. It must query the already-created race_results table (it exists by the time we call this — see Step 3). Do not read S3 or parquet directly; query DuckDB.

Confirmed race_results column names (from scripts/sql_queries.py, create_race_results_query): season, location, year, division, name_raw (athlete name), roxzone_time_min (Roxzone minutes as float), total_time_min. Verify these exist by reading sql_queries.py before writing the query — do not assume.

# module-level constants, near the top with the other constants
INTEGRITY_FANOUT_RATIO = 1.9        # rows/distinct_names at or above this == cross-division fan-out
INTEGRITY_ROXZONE_ZERO_FRAC = 0.95  # fraction of finishers with roxzone == 0
INTEGRITY_MIN_ROSTER = 20           # ignore tiny groups; thresholds are noisy below this


def assert_race_results_integrity(con: duckdb.DuckDBPyConnection) -> list[str]:
    """Return a list of human-readable violations for phantom-duplicate corruption.

    Empty list == safe to build. Each violation names the (season, location, year,
    division) group and which fingerprint it tripped. Read-only: runs SELECTs only.

    Thresholds mirror the upstream scraper's assert_event_integrity. Keep them in sync;
    see hyrox_analysis/scraping_code/hyrox_web_scraping.py.
    """
    rows = con.execute(
        f"""
        WITH grp AS (
            SELECT
                CAST(season AS INTEGER)            AS season,
                trim(CAST(location AS VARCHAR))    AS location,
                CAST(year AS INTEGER)              AS year,
                CAST(division AS VARCHAR)          AS division,
                COUNT(*)                            AS n_rows,
                COUNT(DISTINCT lower(trim(name_raw))) AS n_names,
                AVG(CASE WHEN roxzone_time_min = 0 THEN 1.0 ELSE 0.0 END) AS roxzone_zero_frac
            FROM race_results
            WHERE name_raw IS NOT NULL AND trim(name_raw) <> ''
            GROUP BY 1, 2, 3, 4
        )
        SELECT season, location, year, division, n_rows, n_names, roxzone_zero_frac,
               CAST(n_rows AS DOUBLE) / NULLIF(n_names, 0) AS fanout
        FROM grp
        WHERE n_rows >= {INTEGRITY_MIN_ROSTER}
          AND (
                CAST(n_rows AS DOUBLE) / NULLIF(n_names, 0) >= {INTEGRITY_FANOUT_RATIO}
                OR roxzone_zero_frac >= {INTEGRITY_ROXZONE_ZERO_FRAC}
          )
        ORDER BY season, location, year, division
        """
    ).fetchall()

    violations: list[str] = []
    for season, location, year, division, n_rows, n_names, rox_frac, fanout in rows:
        tells = []
        if fanout is not None and fanout >= INTEGRITY_FANOUT_RATIO:
            tells.append(f"fan-out {fanout:.2f} (rows={n_rows}, distinct_names={n_names})")
        if rox_frac is not None and rox_frac >= INTEGRITY_ROXZONE_ZERO_FRAC:
            tells.append(f"{rox_frac:.0%} zero-roxzone")
        violations.append(
            f"(season={season}, location={location}, year={year}, division={division}): "
            + "; ".join(tells)
        )
    return violations

Step 2 — Add the --allow-dirty override¶

The default must be fail-closed. Provide one escape hatch so a known-imperfect snapshot can still be built deliberately (e.g. during the upstream backfill window).

Two acceptable mechanisms — pick the one that matches the repo's existing entrypoint style (this script currently has no argparse and is invoked as if __name__ == "__main__": ingest_full_refresh()):

  • Preferred (matches current style): an env var read via the existing os.getenv pattern, e.g. ALLOW_DIRTY_INGEST truthy → skip the abort but still log every violation at WARNING. Thread it in as a parameter ingest_full_refresh(allow_dirty: bool = False) defaulting from os.getenv("ALLOW_DIRTY_INGEST", "").strip().lower() in {"1", "true", "yes"}.
  • If the repo already uses argparse elsewhere for scripts, add --allow-dirty instead.

Do not add argparse machinery beyond what is needed for this one flag.

Step 3 — Wire the gate into ingest_full_refresh¶

Call the check immediately after race_results is created and start-dates are applied, and before the athletes / athlete_results / athlete_index / rankings / percentiles / history tables are built. That ordering guarantees we never spend work materializing downstream tables on top of corrupt rows, and that the abort happens inside the open transaction (which is rolled back / the temp DB simply discarded — we never os.replace into place).

Insert right after line ~355 (apply_event_start_dates(con, start_dates)):

    violations = assert_race_results_integrity(con)
    if violations:
        header = (
            f"Integrity gate FAILED: {len(violations)} event group(s) show the "
            f"phantom-duplicate fingerprint (fan-out and/or zero-roxzone)."
        )
        for v in violations:
            logger.warning("  %s", v)
        if not allow_dirty:
            con.execute("ROLLBACK;")
            con.close()
            raise SystemExit(
                header
                + "\nRefusing to build DuckDB. Re-scrape/clean these groups upstream, "
                + "or set ALLOW_DIRTY_INGEST=1 to build anyway (NOT recommended)."
            )
        logger.warning("%s ALLOW_DIRTY_INGEST set -- building anyway.", header)

Make sure no partial DB is swapped into place on abort: the os.replace(...) at the end must not be reached when we raise SystemExit. Because the temp file is config["duckdb_tmp_path"] (the .new file) and the swap is the last step, simply raising before it is sufficient — but explicitly con.close() first so the temp file handle is released. Optionally delete the temp .new file on abort for cleanliness.

Step 4 — Tests¶

Create tests/test_ingest_build_gate.py. Follow the style of the existing tests/test_ingest_start_dates.py. Build a tiny in-memory or temp-file DuckDB with a hand-made race_results table (you do not need S3 — just con.execute("CREATE TABLE race_results (...)") and insert rows) and assert:

  1. Clean data passes — a group of ≥20 distinct athletes, fan-out ~1.0, non-zero roxzone → assert_race_results_integrity(con) returns [].
  2. Fan-out is caught — duplicate every athlete row across two divisions so rows/distinct_names ≥ 1.9 → returns a violation mentioning "fan-out".
  3. Zero-roxzone is caught — ≥20 finishers all with roxzone_time_min = 0 → returns a violation mentioning "zero-roxzone".
  4. Small groups are ignored — a corrupt-looking group with < INTEGRITY_MIN_ROSTER rows returns [] (no false positive on tiny brackets).
  5. Gate aborts by default / honours allow_dirty — either call ingest_full_refresh with a stubbed config, or (simpler and sufficient) unit-test the decision: violations + allow_dirty=False raises SystemExit; violations + allow_dirty=True does not. If wiring a full ingest_full_refresh is too heavy to stub, factor the abort decision into a tiny pure helper and test that directly.

Only race_results columns referenced by the query need to exist in the test table: season, location, year, division, name_raw, roxzone_time_min. Keep fixtures minimal.

Step 5 — Document¶

Add a short note to docs/duckdb-stabilization.md (or docs/data-model.md, whichever covers the ingest): the build now refuses to publish if any event group shows the phantom-duplicate fingerprint; override with ALLOW_DIRTY_INGEST=1. One paragraph, link back to the upstream docs/phantom-duplicate-races-rootcause.md in hyrox_analysis.

Constraints / do-nots¶

  • Read-only. The check runs SELECTs only. Never UPDATE/DELETE rows to "fix" data.
  • No new dependencies. Use duckdb, stdlib, and the existing logging setup.
  • Match existing style in ingest_duckdb_from_s3.py (module-level functions, the logger, the env-var/required_env pattern). Do not refactor unrelated code.
  • Thresholds are a copied contract. They mirror the upstream scraper constants. Add the comment pointing back so a future maintainer keeps them in sync; do not invent different numbers.
  • Fail-closed by default. --allow-dirty / ALLOW_DIRTY_INGEST is the only bypass.
  • Do not touch DB_LOCATION_ALIASES, the start-date logic, or the table-build SQL.

Success criteria¶

  • [ ] assert_race_results_integrity(con) exists, is read-only, returns list[str].
  • [ ] Gate is called after race_results/start-dates, before downstream tables.
  • [ ] Default run aborts (no DB swap) when violations exist; ALLOW_DIRTY_INGEST=1 logs and proceeds.
  • [ ] tests/test_ingest_build_gate.py covers clean-pass, fan-out, zero-roxzone, small-group-ignored, and the abort/allow-dirty decision. All pass.
  • [ ] Existing test suite (pytest -q) stays green.
  • [ ] One paragraph of docs added.

Out of scope¶

  • Cleaning/quarantining existing corrupt S3 partitions — that is the upstream audit script hyrox_analysis/scraping_code/audit_phantom_s3.py (#19).
  • Switching identity to column-based grouping / retiring DB_LOCATION_ALIASES (#18 in the upstream plan is a different downstream change; do not conflate).
